# E1.9 — Email/SMTP & Deliverability

- **Phase:** 1 — Platform foundations
- **Status:** TODO
- **Depends on:** E1.4
- **Blocks:** E3.2 (mail surface), E4.1 (recruiter invite)

## Goal
Outbound mail that lands.

## Scope
SPF/DKIM/DMARC and the outbound path.

## Key insight (scope-narrowing)
Only **one** email needs real-world deliverability — the invite to the recruiter's actual
address. Everything *inside* the company is pre-seeded internal mail. So scope the hard
SPF/DKIM/DMARC work to that single outbound message, not a fully deliverable mail system.

## Deliverable
A reliable outbound path for the single recruiter-invite email (correct SPF/DKIM/DMARC, stays out
of spam), plus the internal-mail seeding approach for everything else.

## Definition of Done
- [ ] SPF/DKIM/DMARC correct for the sending domain.
- [ ] Recruiter-invite email reliably lands in a real inbox (tested).
- [ ] Internal/pre-seeded mail path defined (no external deliverability needed).
