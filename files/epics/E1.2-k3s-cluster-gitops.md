# E1.2 — k3s Cluster Architecture & GitOps  ⭐ FIRST WAVE  · LOCKED: k3s

- **Phase:** 1 — Platform foundations
- **Status:** DONE
- **Depends on:** E1.1 (needs a provisioned node)
- **Blocks:** most app + provisioning epics

## Goal
Define the per-company k3s topology, Helm packaging, and GitOps approach.

## Scope
- One k3s per company on a small VPS.
- Helm charts per app.
- How a deploy is triggered and reconciled (Argo CD / Flux).

## Deliverable
The per-company k3s topology, the Helm packaging convention, and the GitOps reconciliation model
(which tool, repo structure, how a CSV-triggered deploy reaches the cluster).

## Notes
- k3s chosen because it's isolated-per-company, cheap (runs on a $5–20 VPS), provider-agnostic
  (any VM), and still "Kubernetes."
- Provision against a **warm** node (E1.1) so deploys are minutes, not cold-start hours.
