# E2.3 — Seed-Data Generator

- **Phase:** 2 — Provisioning engine
- **Status:** DONE
- **Depends on:** E2.1 (CSV + engine), E3.1 (Files), E3.2 (mail/calendar)
- **Blocks:** the "feels alive" demo experience

## Goal
Beat the empty-shell problem; make a fresh company read as a real one.

## Scope
From the CSV, generate a welcome email, a couple of calendar events, an onboarding doc in Files,
and a few contacts. Enough texture that the recruiter opens it and it feels alive.

## Deliverable
The seed-data spec + generator design — what gets created in which app, how it's templated off
the CSV (company name, employees, roles), and how it's injected idempotently during provisioning.

## Definition of Done
- [x] Seed-data spec defined (which artefacts in which app).
- [x] Generator templates off CSV fields.
- [x] Injection is idempotent.
- [x] A freshly provisioned company reads as populated, not empty.
