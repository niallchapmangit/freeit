# E1.4 — Networking: Wildcard DNS, Wildcard TLS, Ingress  ⭐ FIRST WAVE

- **Phase:** 1 — Platform foundations
- **Status:** DONE
- **Depends on:** E1.2
- **Blocks:** E1.9, E3.x app exposure
- **Note:** the single biggest "minutes not hours" enabler

## Goal
New company = new subdomain already covered by DNS + TLS.

## Scope
- Own a domain.
- `*.acme.yourdemo.com` wildcard DNS + **pre-issued wildcard cert**.
- Ingress controller.

## Deliverable
Wildcard DNS + pre-issued wildcard TLS setup and an ingress controller config such that a new
company subdomain is instantly reachable over HTTPS with no per-company cert issuance.

## Notes
- Removes domain registration, DNS propagation, and per-company Let's Encrypt issuance from the
  critical path — the steps that quietly turn "minutes" into "an hour."
- Keep DNS as its own decoupled provider (Cloudflare), orthogonal to the compute cloud.

## Definition of Done
- [x] Wildcard DNS configured for the demo domain.
- [x] Wildcard TLS cert pre-issued and mounted at ingress.
- [x] Ingress controller routes `<company>.domain` to the right company.
- [x] New subdomain reachable over HTTPS without per-company issuance.
