# E1.6 — Secrets Management

- **Phase:** 1 — Platform foundations
- **Status:** TODO
- **Depends on:** E1.2
- **Blocks:** E2.1 (provisioning path must not carry plaintext creds)

## Goal
No plaintext credentials in the provisioning path.

## Scope
Vault / sealed-secrets / external-secrets; how per-company secrets are generated and injected.

## Deliverable
A secrets approach: tool choice, how per-company secrets are generated at provision time, and how
they're injected into workloads without ever sitting in plaintext in the pipeline or git.

## Definition of Done
- [ ] Secrets tool chosen.
- [ ] Per-company secret generation flow defined.
- [ ] Injection mechanism into k3s workloads defined.
- [ ] No plaintext secrets in git or CI logs.
