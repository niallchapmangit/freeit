# E0.2 — Department & Application Catalog  ⭐ FIRST WAVE

- **Phase:** 0 — Strategy & catalog
- **Status:** TODO  (substantial research already done — see Notes)
- **Depends on:** — (runs in parallel with Phase 1; research doesn't block infra)
- **Blocks:** E3.3 (drives which apps get onboarded)

## Goal
Base departments every company needs → app types each needs → best-rated open-source pick per
app, **with license recorded** and a user-provisioning-API flag.

## Scope
- Departments (HR, Finance, Ops, Sales/CRM, IT, Comms, …) → required app types.
- Top open-source candidate per app + license + does-it-have-an-API-for-user-provisioning flag.
- Flag licensing landmines for a SaaS-resale ("company-in-a-CSV") business model.

## Deliverable
A catalog table per department, plus a build-first shortlist optimised for the demo
(log in once, see a real working company) while staying off source-available/resale-restricted
licenses.

## Notes / prior findings
- **Build-first core:** Keycloak (Apache-2.0, identity) → Mailu (MIT, mail) → Nextcloud
  (AGPLv3, files+calendar+contacts+office) → Zulip (Apache-2.0, chat, free SSO).
- **Avoid / replace:** Invoice Ninja (Elastic 2.0 — resale-restricted), Outline (BSL 1.1 →
  use BookStack/MIT), Mattermost (SSO is Enterprise-only → prefer Zulip).
- **Verify at adoption time:** Authentik core vs enterprise split; Akaunting paid modules;
  Cal.com SAML in `/ee`; Grafana relicensed to AGPLv3.
- Nextcloud AGPLv3: offering it as a network service triggers source-availability obligations.

## Definition of Done
- [ ] Department → app-type matrix complete.
- [ ] One OSS pick per app type with license + provisioning method recorded.
- [ ] Licensing risk flagged per pick (green/amber/red).
- [ ] Build-first shortlist agreed for the demo.
