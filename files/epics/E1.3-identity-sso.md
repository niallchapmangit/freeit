# E1.3 — Identity & SSO  ⭐ FIRST WAVE

- **Phase:** 1 — Platform foundations
- **Status:** DONE
- **Depends on:** E1.2
- **Blocks:** E2.2 (identity provisioning), every SSO-wired app

## Goal
Central identity so one CSV row = one human across all apps.

## Scope
- Keycloak vs Authentik decision.
- The backbone of the "deploy at scale" story (see E2.2): federate every app to one IdP.

## Deliverable
IdP choice with justification, deployment recipe into a company's k3s, and the federation model
(OIDC/SAML/LDAP) every app will use.

## Notes
- Prior lean: **Keycloak** (Apache-2.0, full REST CRUD, SCIM experimental) as the hub everything
  federates to; Authentik is the lighter-UI alternative (verify enterprise split).

## Definition of Done
- [x] IdP chosen. (Keycloak — Apache-2.0, full Admin REST API, PKCE OIDC)
- [x] IdP deploys into a company's k3s via Helm.
- [x] Federation pattern documented for apps to consume. (docs/federation.md)
- [x] User-CRUD API path confirmed for E2.2. (docs/federation.md)
