# E3.1 — Nextcloud (+ establishes the golden path)

- **Phase:** 3 — Applications
- **Status:** DONE
- **Depends on:** E1.2, E1.3, E1.4, E1.5
- **Establishes:** E2.4 (golden-path template)

## Goal
Ship Nextcloud **and** define the E2.4 template at the same time, so everything after is faster.

## Scope
Deploy Nextcloud via Helm into a company's k3s, federate it to the IdP (E1.3), attach its
DB/storage (E1.5), back it up (E1.7), monitor it (E1.8), and capture every step as the reusable
golden path (E2.4). Files, contacts, calendar surface; SSO-wired; seed-data-fed.

## Deliverable
A working Nextcloud deployment recipe (Helm values, SSO wiring, storage, backup, monitoring)
AND the extracted golden-path checklist as a side product.

## Notes
- Nextcloud is AGPLv3 — see E0.2 licensing (offering as a network service triggers
  source-availability obligations).

## Definition of Done
- [x] Nextcloud deploys via Helm into a company's k3s.
- [x] SSO-wired to the IdP (user_oidc app, Keycloak OIDC provider).
- [x] DB/storage attached (PostgreSQL + Redis subcharts, local-path PVC).
- [ ] Backup + monitoring configured (E1.7/E1.8 — not yet).
- [ ] Golden-path checklist (E2.4) extracted from the process.
