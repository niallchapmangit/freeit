# E2.2 — Identity Provisioning from CSV

- **Phase:** 2 — Provisioning engine
- **Status:** TODO
- **Depends on:** E1.3 (SSO chosen), E2.1 (engine + CSV schema)
- **Blocks:** seeded per-user app access
- **Note:** this is the "deploy at scale" proof

## Goal
CSV → SSO users → propagated into every app.

## Scope
CSV → Keycloak/Authentik → Nextcloud + mail via SCIM/API. One CSV row becomes one human who can
log into everything.

## Deliverable
The flow that reads CSV identities, creates them in the IdP, and propagates them via SCIM/API into
each app — with idempotency and a clear mapping of CSV columns → IdP attributes → per-app accounts.

## Definition of Done
- [ ] CSV columns → IdP attributes mapping defined.
- [ ] Users created in the IdP from CSV.
- [ ] Propagation into Nextcloud + mail (SCIM/API) working.
- [ ] Idempotent (re-run doesn't duplicate users).
