# E1.1 — IaC & Provider-Agnostic Provisioning  ⭐ FIRST WAVE

- **Phase:** 1 — Platform foundations
- **Status:** TODO  (design work started — see Notes)
- **Depends on:** — (foundation)
- **Blocks:** E1.2 (k3s needs a node) and everything downstream

## Goal
Terraform/OpenTofu that stands up a node + base infra, replicable beyond AWS.

## Scope
VM provisioning, networking primitives, state management. This epic is where "provider
agnostic" actually lives — keep provider-specific bits behind a thin interface so the same plan
targets AWS first and another VPS later.

## Deliverable
A Terraform/OpenTofu layout (modules, state strategy, provider abstraction) that provisions one
VM + networking on AWS, with the seams identified for swapping providers. Propose module
structure and state approach before writing HCL.

## Notes / prior findings
- **"Provider agnostic" is a contract, not a framework.** One documented input/output contract;
  one module per provider that implements it; portable layer (cloud-init, k3s install, app)
  lives above the seam. Discipline enforced by review, not tool conditionals.
- **Unit of provisioning = one company = one isolated state.** Do not use one giant state.
- **State:** OpenTofu ≥ 1.10 — S3 backend with `use_lockfile` (drop DynamoDB) + **native
  client-side state encryption** (GA since 1.7, AES-GCM, KMS key in EU account) = the GDPR lever.
- **Bootstrap chicken-and-egg:** a tiny `bootstrap/` stack with local state creates the
  encrypted, versioned, EU-region state bucket (run once per cloud account).
- **DNS is a separate, decoupled provider** (recommend Cloudflare) — orthogonal to compute.
- **Firewall can cross the seam:** SG on AWS vs on-host ufw/iptables via cloud-init on a bare VPS;
  keep the port list as the contract, let each substrate decide enforcement.

## Definition of Done
- [ ] `bootstrap/` stack creates encrypted, versioned, EU-region state bucket.
- [ ] Provider contract (inputs/outputs) documented.
- [ ] AWS substrate module provisions 1 VM + networking + stable IP + firewall.
- [ ] State encrypted client-side; `required_version = ">= 1.10"` pinned.
- [ ] Seam map documented (what swaps, and at what cost, per provider).
