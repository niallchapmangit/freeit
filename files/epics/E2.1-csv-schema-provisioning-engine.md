## Scope
- Define the CSV contract (company, domain, employees, roles, …).
- The orchestrator that turns a row into: create instance → deploy charts → create SSO users

- **Must be idempotent and re-runnable** (E4.2's fallback + live run rely on this:
  re-running the same CSV converges, doesn't duplicate).

## Deliverable
The CSV schema definition and the orchestrator design — the step pipeline
(instance → charts → SSO users → seed) with idempotency baked in. Propose the architecture
before code.
